# N8N-Ollama 플랫폼 아키텍처 문서

## 시스템 개요

N8N-Ollama 플랫폼은 오픈소스 워크플로우 자동화 엔진인 N8N과 로컬 LLM 추론 엔진인 Ollama를 Docker 기반으로 통합한 AI 자동화 플랫폼입니다.

---

## 서비스 구성

| 서비스     | 이미지              | 포트 | 역할                        |
|-----------|---------------------|------|-----------------------------|
| N8N       | n8nio/n8n:latest    | 5678 | 워크플로우 자동화 엔진        |
| Ollama    | ollama/ollama:latest | 11434 | 로컬 LLM 추론 서버          |
| Redis     | redis:7-alpine      | 6379 | 세션 캐시 및 대화 기록       |
| PostgreSQL| postgres:16-alpine  | 5432 | 영구 데이터 저장              |

---

## 데이터 흐름

### 메인 AI 어시스턴트 (Workflow 1)
1. 클라이언트 → N8N 웹훅 (API 키 인증)
2. N8N → Redis (대화 기록 조회)
3. N8N → Ollama (llama3.2 모델로 추론)
4. N8N → Redis (업데이트된 기록 저장)
5. N8N → PostgreSQL (전체 대화 로그)
6. N8N → 클라이언트 (응답 + 메타데이터)

### 멀티 모델 라우터 (Workflow 2)
1. 클라이언트 → N8N 웹훅 (메시지 + 복잡도 힌트)
2. N8N 복잡도 분석 (키워드 + 길이 기반)
3. 라우팅: simple → llama3.2:1b / medium → llama3.2 / complex → llama3.2:latest
4. N8N → PostgreSQL (라우팅 결정 로그)
5. N8N → 클라이언트 (응답 + 선택 이유)

### 감정 분석 (Workflow 4)
1. 클라이언트 → N8N 웹훅
2. N8N → Ollama (감정 분석, JSON 형식 응답 요청)
3. N8N 감정 파싱 (positive/negative/neutral + 신뢰도)
4. 톤 선택 (긍정→열정적, 부정→공감적, 중립→정보 제공)
5. N8N → Ollama (선택된 톤으로 최종 응답 생성)
6. N8N → PostgreSQL (감정 분석 로그)
7. N8N → 클라이언트 (응답 + 감정 메타데이터)

## 보안 아키텍처

- **API 키 인증**: 모든 웹훅에 `X-API-Key` 헤더 필수
- **네트워크 격리**: `ollama-network` Docker 브리지 네트워크로 서비스 간 격리
- **환경 변수**: `.env` 파일로 민감 정보 분리 (Git에서 제외)
- **Redis 인증**: requirepass 설정으로 비밀번호 보호
- **PostgreSQL 인증**: 전용 사용자/비밀번호로 데이터베이스 보호

## 성능 고려사항

| 항목           | 설정값              | 이유                          |
|---------------|---------------------|-------------------------------|
| N8N 메모리     | 최대 1GB            | 다수 워크플로우 동시 실행      |
| Ollama 메모리  | 최대 8GB            | LLM 모델 로드를 위한 충분한 메모리 |
| Redis 메모리   | 최대 256MB          | LRU 정책으로 세션 캐시 관리    |
| 세션 TTL       | 86400초 (24시간)    | 하루 단위 대화 유지           |
| 최대 기록 수   | 20개 메시지         | 컨텍스트 윈도우 과부하 방지    |

## 모델 선택 기준 (Workflow 2)

```
단순 (simple)   → llama3.2:1b
  조건: 단순 키워드 포함 AND 단어 수 ≤ 10
  특징: 빠른 응답, 낮은 메모리 사용

중간 (medium)   → llama3.2
  조건: 단순/복잡 기준에 해당하지 않는 경우
  특징: 균형 잡힌 성능

복잡 (complex)  → llama3.2:latest
  조건: 복잡 키워드 포함 OR 단어 수 > 50 OR 글자 수 > 300
  특징: 최고 품질 응답, 더 긴 처리 시간
```
